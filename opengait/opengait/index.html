Model Training

Prepare dataset

See prepare dataset.

Get trained model

  • Option 1: 
    python misc/download_pretrained_model.py
  • Option 2: Go to the release page, then download the model file and uncompress it to output.

Train

Train a model by 

CUDA_VISIBLE_DEVICES=0,1 python -m torch.distributed.launch --nproc_per_node=2 opengait/main.py --cfgs ./config/baseline/baseline.yaml --phase train
- python -m torch.distributed.launch DDP launch instruction. - --nproc_per_node The number of gpus to use, and it must equal the length of CUDA_VISIBLE_DEVICES. - --cfgs The path to config file. - --phase Specified as train.

  • --log_to_file If specified, the terminal log will be written on disk simultaneously.

You can run commands in train.sh for training different models. Training is done on a Linux environment. Commands and code would have to be changed manually according to your specific OS environment.

Models We Created

To run the models we created, a Gait Ensemble of GaitSet + GaitPart and a Hybrid model, use these commands:

Gait Ensemble 

CUDA_VISIBLE_DEVICES=0 python -m torch.distributed.launch --nproc_per_node=2 opengait/main.py --cfgs ./config/gaitens.yaml --phase train
Hybrid 
CUDA_VISIBLE_DEVICES=0 python -m torch.distributed.launch --nproc_per_node=2 opengait/main.py --cfgs ./config/hybrid.yaml --phase train

Test

Evaluate the trained model by 

CUDA_VISIBLE_DEVICES=0,1 python -m torch.distributed.launch --nproc_per_node=2 opengait/main.py --cfgs ./config/baseline/baseline.yaml --phase test
--phase : Specified as test ,--iter : Specify a iteration checkpoint

Tip: Other arguments are the same as train phase.

You can run commands in test.sh for testing different models.

Customize

Warning

  • In DDP mode, zombie processes may be generated when the program terminates abnormally. You can use this command sh misc/clean_process.sh to clear them.
